# Kids Educational App Dashboard UI Skill

## When to use
- When reviewing dashboards or main screens for kids educational apps
- Especially Quran, Arabic, or Islamic learning apps

## User Context
- Target user: Children (ages 4â€“9)
- Secondary user: Parents
- Goal: Learning through simplicity and joy

## What to analyze
1. Visual clarity (colors, spacing, contrast)
2. Cognitive load (avoid overcrowding)
3. Emotional comfort (friendly, safe feeling)
4. Navigation simplicity
5. Motivation elements (progress, rewards)

## UI Rules
- Use large cards, not dense lists
- One main action per screen
- Avoid sharp colors or aggressive contrasts
- Icons + text always together
- Animations must be slow and soft

## Dashboard Improvements Checklist
- Clear welcome message
- Progress visualization (stars, levels, circles)
- Easy resume learning button
- Minimal text, maximum clarity
- No hidden actions

## Output format
- UI Problems (short bullets)
- UX Improvements (step-by-step)
- Visual Suggestions (colors, layout)
- Optional redesigned layout description
